# Модули MCP

После [подключения](page:connect) AI может работать со следующими модулями. Для каждого приведены примеры запросов — их достаточно просто написать в чат.

## 📋 Задачи (Tasks)

Видит доску, создаёт задачи, перемещает между колонками, назначает ответственных, пишет комментарии.

- «Создай в проекте DEV задачу „Подготовить отчёт" и назначь Али»
- «Покажи мои открытые задачи и скажи, какая самая срочная»

## 📚 Wiki

Читает страницы, ищет по ним, создаёт новые страницы.

- «Найди в Wiki руководство по онбордингу и перескажи кратко»
- «Создай страницу „Памятка для нового сотрудника"»

## 📅 Календарь

Видит, создаёт и изменяет события (в том числе повторяющиеся); отправляет приглашения по номеру телефона — с тем же правилом приватности: имя по номеру не показывается, оно появляется только после принятия приглашения; выдаёт ссылку ICS; видит дедлайны задач.

- «Поставь на завтра на 15:00 событие „Командная встреча" и пригласи +998 90 123 45 67»

## 📁 Файлы

Видит папки, читает текстовые файлы, ищет и по имени, и **по содержимому**.

- «Найди документы, где встречается слово „аренда"»
- «Что лежит в папке с договорами?»

## 📄 Документы EDO / Didox

Список документов, их статусы и детали; данные контрагента по ИНН; поиск кодов MXIK.

- «В каком статусе счета-фактуры, пришедшие в этом месяце?»
- «Какой компании принадлежит ИНН 302936161?»

## 🧾 Налоги (Soliq)

Общая картина, отчёты, платежи, чеки, данные по аренде.

- «Покажи налоговые платежи за июнь»
- «Есть ли несданные отчёты?»

## 👥 Сотрудники (HR)

Сотрудники, отделы, зарплата (payroll), табель, отпуска, праздничные дни.

- «Кто в отпуске в июле?»
- «Выведи список сотрудников IT-отдела»

## 🏢 Компании

Данные компании и общий обзор.

- «Дай краткую сводку по моей компании»

> Админ тенанта в разделе **Настройки → MCP** может отключить любой модуль или перевести его в режим «только чтение» (read-only). Тогда AI не сможет ничего менять в этом модуле.
