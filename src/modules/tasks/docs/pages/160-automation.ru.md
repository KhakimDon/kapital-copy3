# Автоматизация

Доска может сама выполнять повторяющиеся действия. **Автоматизация** — это правила, которые создаются без кода. Они находятся в разделе *[Настройки проекта](page:settings) → **Автоматизация***; правила настраивает владелец/админ проекта.

Каждое правило состоит из трёх частей:

- **КОГДА** (триггер) — при каком событии срабатывает правило;
- **ЕСЛИ** (условия) — необязательные фильтры;
- **ТОГДА** (действия) — что нужно сделать.

## Создание правила

1. В настройках проекта найдите раздел **Автоматизация** и нажмите **Правило**.
2. Введите название правила.
3. **Когда** — выберите триггер:
   - **При создании** — когда появилась новая задача (необязательно: в каком столбце);
   - **При перемещении** — когда карточка перешла из столбца в столбец; можно указать столбцы **Откуда** и **Куда** (или «Любой»);
   - **При назначении** — когда сменился ответственный;
   - **При смене важности** — необязательно: на какую новую важность;
   - **При комментарии** — когда к задаче добавили комментарий.
4. **Если** — при необходимости добавьте условие: поле (**Важность**, **Столбец**, **Тип**, **Ответственный**, **Метка**, **Заголовок**) + условие (**равно**, **не равно**, **пусто**, **не пусто**, **содержит**) + значение. Если условий несколько, должны выполниться все.
5. **Тогда** — добавьте одно или несколько действий:
   - **Переместить в столбец** — переносит карточку в выбранный столбец;
   - **Назначить ответственного** — выбранному пользователю, **Автору** или **Снять**;
   - **Изменить важность**;
   - **Добавить метку** / **Убрать метку**;
   - **Сдвинуть срок** — сдвигает срок на указанное число дней;
   - **Написать комментарий** — автоматический комментарий по шаблону;
   - **Уведомить наблюдателей** — отправляет [уведомление](page:notifications) наблюдателям;
   - **Сообщение в Telegram** — сообщение по шаблону в [подключённую группу Telegram](page:telegram).
6. Нажмите **Сохранить**.

В шаблонах комментария и Telegram работают подстановки: `{{title}}` — заголовок задачи, `{{actor}}` — пользователь, выполнивший действие, `{{priority}}` — важность.

## Включение и отключение

Слева у каждого правила в списке есть переключатель — правило можно временно приостановить, не удаляя. Значок карандаша редактирует, корзина удаляет.

## Примеры

- **При перемещении в «Done»** → **Написать комментарий**: `{{actor}} завершил задачу` + **Сообщение в Telegram**.
- **При создании**, если **Ответственный пусто** → назначить **Автору**.
- **При смене важности** (новая важность: Срочно) → **Добавить метку**: `urgent` + **Уведомить наблюдателей**.

> Правила срабатывают как на изменения в приложении, так и на изменения, сделанные [через AI (MCP)](page:connect). Действие одного правила может запустить другое — цепочка продолжается не более чем на 3 шага.
